cURL

Usefull Options

  • -v : verbose.
    Show both the request and the response.
  • -X <method> : methods selection
    Webserv allows GET, POST, DELETE.
  • -H <header>: to add a header
    e.g. -H "Transfer-Encoding: chunked"
  • -T filepath: add the file's contents to the request's body.
    e.g. -T "file_2048.txt"
  • -d/--data <data>: add the data to the request's body
    Default is Content-Length.
  • --resolve <host:port:address>: provide a custom DNS entry
    e.g. --resolve www.webserv.be:8080:127.0.0.1

Common Configurations

GET

  • Default server with port 8080:
    sh curl -v -X GET http://localhost:8080/
  • www.webserv.be with port 8080:
    sh curl -v -X GET http:// www.webserv.be:8080/ --resolve www.webserv.be:8080:127.0.0.1
  • www.webserv.team with port 8080:
    sh curl -v -X GET http:// www.webserv.team:8080/ --resolve www.webserv.team:8080:127.0.0.1

POST

  • Without any transfer encoding and body's input as a string:
    sh curl -v -X POST http://www.webserv.be:8080/uploads/file.txt --resolve www.webserv.be:8080:127.0.0.1 -d "BODY IS HERE"
  • With chunked transfer encoding and body's input as a file:
    sh curl -v -X POST http://www.webserv.be:8080/uploads/file_2048.txt --resolve www.webserv.be:8080:127.0.0.1 -H "Transfer-Encoding: chunked" -T "./file_2048.txt"

DELETE

  • sh curl -v -X DELETE http://www.webserv.be:8080/delete/delete_me.txt --resolve www.webserv.be:8080:127.0.0.1
